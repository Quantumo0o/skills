# 竞品调研方法论手册（Research Playbook）

> SKILL.md 的延伸参考。包含拆解示例、填写规范、决策树、反模式等深度方法论。
> 仅在需要具体示例或细节时阅读。

---

## 1. 场景拆解方法

将核心场景拆为**可独立观察的最小操作节点**（每节点 = 意图 + 操作 + 反馈）。

**好的拆解**（粒度适中）：

```
注册流程 → 启动APP → 选择注册方式 → 填写信息 → 验证身份 → 完成注册 → 新手引导
```

**坏的拆解**（太粗）：

```
注册流程 → 注册 → 完成
```

**坏的拆解**（太细）：

```
注册流程 → 点击图标 → 等待启动 → 看到闪屏 → 闪屏消失 → 看到首页 → ...
```

每个节点按以下结构分析：

```
节点名称：[例：选择注册方式]
├── 入口：用户从哪里到达？
├── 操作：做什么？（点击/滑动/输入/等待）
├── 反馈：系统反馈？（视觉/动效/文案/震动）
├── 出口：完成后去哪里？
└── 异常：失败/中断/超时怎样？
```

## 2. 对标总览表填写规范

### 列规范

| 列 | 要求 | 好的例子 | 坏的例子 |
|----|------|----------|----------|
| 产品 | 产品名 | 小红书 | 小红书 v8.2 iOS 最新版 |
| 场景/节点 | 动词短语，≤8 字 | 选择注册方式 | 用户在首次打开App时需要选择一种注册方式 |
| 做法与亮点 | 2-3 句，用「→」连接因果 | 底部 Tab 居中 FAB，点击展开类型选择器 → 1 次点击直达编辑器 | 有个加号按钮 |
| 与我方差异 | 竞品 vs 我方 → 差距 | 竞品支持草稿自动保存，我方需手动保存 → 创作中断流失风险 | 他们比我们好 |
| 来源 | SRC-xxx | SRC-01 | 无 |
| 判定 | 🔴/🟡/🟢/⚪ | 🟡 明显差异 | 一般 |

### 信息密度标准

- "做法与亮点"列：50-120 字
- "与我方差异"列：30-80 字
- 无我方锚点时：将"与我方差异"改为"行业差异化价值"

## 3. 严重度决策树

```
该差距是否出现在核心路径（D1/D2）上？
├── 是 → 是否导致用户无法完成核心任务？
│   ├── 是 → 🔴 致命差距
│   └── 否 → 是否可被 ≥30% 用户明显感知？
│       ├── 是 → 🟡 明显差异
│       └── 否 → ⚪ 持平
└── 否 → 是否影响非核心但重要的体验？
    ├── 是 → 🟡 明显差异
    └── 否 → ⚪ 持平

我方做法明显优于所有竞品？ → 🟢 我方领先
```

## 4. 关键发现完整示例

> **[D5 · 激励/增长]** 新创作者流量扶持机制缺失
>
> **竞品做法**：Instagram 为新创作者的前 10 条 Reels 提供独立推荐池，保底 500+ 曝光；小红书对新号前 5 条笔记给予流量倾斜，发布 2 小时内优先分发。
>
> **我方现状**：新用户发帖后无任何流量扶持，完全依赖自然推荐，首帖平均曝光 <50。
>
> **差距分析**：缺少冷启动扶持导致"发了没人看→不想发"恶性循环，直接影响创作者 7 日留存率。
>
> 🔴 致命差距 · SRC-02 🔗公开信息

## 5. Pattern 描述结构

| 要素 | 要求 | 说明 |
|------|------|------|
| 名称 | 动词/名词短语，≤10 字 | 如"居中 FAB 发布入口" |
| 适用场景 | 哪类产品/功能可用 | 不要写"所有产品" |
| 交互规则 | 操作方式、触发条件、状态变化 | 具体到点击/滑动/长按 |
| 视觉规则 | 尺寸、颜色、层级、动效 | 给参考数值 |
| 边界/风险 | 不适用场景、副作用 | 如"Tab ≥6 个时空间不足" |
| 来源产品 | 哪些竞品使用此模式 | 至少列 2 个 |

## 6. 落地建议反模式

| 坏的建议 | 为什么坏 | 好的建议 |
|----------|---------|----------|
| "优化发帖体验" | 没说做什么 | "发布入口改为底部 Tab 居中 FAB，点击直达编辑器" |
| "参考竞品做法" | 没说参考哪个 | "参考小红书草稿自动保存机制，退出时自动存草稿" |
| "提升转化率" | 只说目标没说手段 | "首次发帖引导：新用户进入编辑器时展示 3 个话题模板" |
| "后续跟进" | 没有时间和Owner | "产品经理 T+7 输出发帖漏斗数据报告" |

## 7. 信息来源管理

所有来源编号 `SRC-xxx`，按以下格式登记：

```
SRC-001  📸用户材料  小红书 · iOS · 发帖流程截图（用户提供）
         URL: N/A（用户本地截图）

SRC-002  🔗公开信息  Instagram · 官方博客 "Reels Best Practices"
         URL: https://about.instagram.com/blog/announcements/reels-best-practices

SRC-003  💡行业经验  社交产品 UGC 发帖流程通用认知
         URL: N/A（行业通识，建议实际验证）
```

**引用规则**：

- 每条关键发现至少关联 1 个 SRC
- 对标总览表每行至少关联 1 个 SRC
- 预期影响含数据时，必须关联来源或标注 `[估算]`
- 💡行业经验结论文末加注"建议实际验证"

**可信度排序**：📸 用户材料 > 🔗 官方来源 > 🔗 第三方来源 > 💡 行业经验

## 8. 聚焦度诊断

### 散焦判定（满足任一 = 过于宽泛）

1. **模块跨度过大**：覆盖 ≥3 个独立子模块
2. **缺少痛点/动机**：只说"看看行业怎么做的"
3. **竞品形态冲突**：对标产品设计哲学根本冲突（如强视觉社区 vs 长文问答社区）

### 散焦时回复模板

```
⚠️ 聚焦度提示：您的调研目标覆盖了 X 个独立子模块，建议聚焦 1-2 个：

  A. [子课题 A]（如：发帖入口位置与层级设计）
  B. [子课题 B]（如：媒体编辑器体验对比）
  C. [子课题 C]（如：草稿保存与异常恢复机制）

请选择，或回复"全部保留"（报告将标注为"基建扫盲"模式）。
```

### 缺少锚点时回复模板

```
💡 锚点建议：您是否有正在开发/运营的产品？
  - 如有：请简述当前做法和痛点，报告将生成"差距分析+落地建议"
  - 如无：报告将以"行业基建标配"模式输出
```

### 报告模式

| 条件 | 模式 | 总览表差异列 | 封面标签 |
|------|------|------------|---------|
| 有锚点 + 聚焦 | 🎯 精准诊断 | "与我方差异" | 精准对标 |
| 有锚点 + 宽泛 | 📋 全景扫描 | "与我方差异" | 全景扫描 |
| 无锚点 + 聚焦 | 🔍 行业深挖 | "行业差异化价值" | 行业深挖 |
| 无锚点 + 宽泛 | 📖 基建扫盲 | "行业差异化价值" | 基建扫盲 |

## 9. 竞争力健康度计算

仅在有我方锚点时计算（精准诊断 / 全景扫描模式）。无锚点模式不显示。

### 计算示例

假设报告有 6 条关键发现：

| # | 维度 | 严重度 | 基础分 | 权重 | 加权分 |
|---|------|--------|--------|------|--------|
| 1 | D1 信息架构 | 🟡 明显差异 | 35 | 1.5 | 52.5 |
| 2 | D2 交互流程 | 🔴 致命差距 | 0 | 1.5 | 0 |
| 3 | D3 视觉设计 | ⚪ 持平 | 70 | 1.0 | 70 |
| 4 | D5 激励/增长 | 🔴 致命差距 | 0 | 1.0 | 0 |
| 5 | D6 异常/边界 | 🟢 我方领先 | 100 | 1.0 | 100 |
| 6 | D8 合规/无障碍 | ⚪ 持平 | 70 | 1.0 | 70 |

```
权重总和 = 1.5 + 1.5 + 1.0 + 1.0 + 1.0 + 1.0 = 7.0
加权总分 = 52.5 + 0 + 70 + 0 + 100 + 70 = 292.5
健康度 = 292.5 ÷ 7.0 = 41.8 → 42 分 → C 级（竞争力告急）
```

### 等级映射

| 分数 | 等级 | 徽章标签 | 含义 |
|------|------|---------|------|
| 85-100 | S | 竞争力领先 | 在对标维度上全面领先或持平，无致命差距 |
| 70-84 | A | 竞争力健康 | 整体健康，有少量可感知差距但无断裂 |
| 55-69 | B | 存在差距 | 多个维度落后，需有针对性补强 |
| 0-54 | C | 竞争力告急 | 核心路径存在致命缺陷，建议优先修复 |

### 校验规则

- 存在 ≥1 个 🔴 致命差距时，等级不可为 S
- 所有发现均为 🟢/⚪ 时，等级不可低于 A
