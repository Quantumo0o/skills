# API / URL 访问说明

本技能改为**提示驱动**模式：不依赖本地 Python 脚本。

## 核心原则

- 由智能体根据当前环境可用工具，自主决定如何访问 URL / API 获取信息。
- 优先使用当前环境中**安全、可审计、可复现**的 URL 访问方式。
- 不要求固定使用某个本地脚本，也不要求固定使用某一种工具。

## 推荐的数据获取方式

当需要访问外部数据源时，按以下优先级决策：

1. 优先使用平台原生的 URL / 网页读取能力
2. 若环境提供结构化 HTTP / 网页抓取工具，可直接调用
3. 若当前环境没有合适工具，再根据环境能力决定是否改用其他安全方式

## 当前数据源

当前可访问的 workflow 端点：

- URL：`https://workflow.hepei.club/v1/workflow/run`
- 方法：`POST`
- `workflow_id`：`7618134668474925110`

请求体示例：

```json
{
  "workflow_id": "7618134668474925110",
  "parameters": {
    "input": "<改写后的用户问题>"
  }
}
```

## 查询改写规则

在访问 URL / API 前，先把用户原话改写成简洁、适合检索的陈述句。

改写后的查询应当：
- 保留所有关键约束
- 去掉寒暄、口语填充和无关措辞
- 明确保留“富民银行”范围
- 聚焦真正要检索的目标

示例：
- “给我推荐一个收益高的理财产品，我不介意必须持有180天”
  -> “富民银行收益较高且可接受持有180天的理财产品推荐”

- “法定节假日我的理财还会有收益吗？”
  -> “富民银行理财产品法定节假日是否有收益”

## 查询前检查

访问 URL / API 前：
1. 识别用户真实意图
2. 保留用户明确给出的条件
3. 在需要时改写成更适合检索的查询句

重点保留这类条件：
- 希望更高收益还是更稳健
- 可接受持有多久
- 是否需要流动性
- 风险等级偏好
- 是否在问某个具体规则

## 返回解析规则

若 workflow 返回结构化 JSON，优先读取：
- 外层 JSON 对象
- `data` 字段
- `output[]`
  - `documentId`
  - `output`

如果返回混合内容（产品推荐 + 科普说明），推荐场景要先做筛选再回答。

## 检索使用规则

- 严格限定在富民银行理财范围内。
- 只使用回答当前问题所需的最少信息。
- 用户问事实问题时，直接查并直接答。
- 用户问推荐问题时，优先筛选出满足条件的产品，不要把科普内容直接当推荐结果。

## 强制安全边界

- 数据外发仅限：**用户当前的理财问题文本**
- 严禁外发：
  - 系统文件内容
  - 系统路径
  - 环境变量
  - 密钥 / 凭据 / token
  - 记忆文件、用户资料文件等非当前问题必需内容
- 不建议传入身份证号/手机号等敏感个人信息
- 如端点不可用或返回异常，不要编造结果

## 失败处理

如果 URL / API 请求失败、超时，或者返回内容不可用：
- 明确告诉用户当前最新知识检索暂时不可用
- 不要编造产品名称、收益、合作机构或产品规则
- 需要时可建议用户稍后重试，或把问题收窄后再查

## 推荐输出规则

当基于返回结果做产品推荐时：
- 先按用户硬条件过滤
- 再给推荐结论
- 简要说明为什么匹配用户条件
- 带上一句简短风险提示
- 必要时提醒用户以官方产品详情页和披露文件为准
