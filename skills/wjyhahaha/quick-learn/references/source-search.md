# 权威资料搜索策略

## 搜索维度优先级

### 1. 官方文档（最高优先级 ⭐⭐⭐）
- 搜索词: `{topic} documentation site:官方域名` 或 `{topic} docs`
- 验证: URL 必须是官方域名或官方 GitHub 组织页
- 标记为「必读」
- **时效**: 检查文档版本号，确保是当前稳定版

### 2. 权威教程/书籍（⭐⭐⭐）
- 搜索词: `{topic} tutorial OR guide OR "getting started" 2024..2026`
- 优先: 知名作者、高 star GitHub 项目、知名平台
- **时效验证**: 发布日期 < 2 年，或明确标注兼容最新版本
- **权威验证**: 检查作者背景（官方团队成员/知名开发者/大学教授）

### 3. 知名技术社区（⭐⭐）
- 搜索词: `{topic} 深入 OR 实践 OR 踩坑`（中文）/ `{topic} in-depth OR deep-dive`（英文）
- 平台: freeCodeCamp、MDN、Dev.to、掘金高赞、少数派
- 帖子需有明确作者、高阅读量/star 数、有评论区互动

### 4. 社区精选/Roadmap（⭐⭐）
- 搜索词: `awesome {topic}` / `{topic} roadmap 2024` / `{topic} learn path`
- 作为学习路径规划的参考

### 5. 视频/音频（⭐⭐）
- 搜索词: `{topic} tutorial video` / `{topic} course playlist`
- 优先 YouTube、B站官方频道、知名博主
- 检查上传日期 < 1 年（技术内容更新快）

### 6. 论文/深度文章（⭐）
- 搜索词: `{topic} survey OR paper OR review`
- 仅对进阶用户推荐

## 权威验证清单（必须逐项检查）

对每个候选材料快速检查：
- [ ] **来源可信**: 官方/知名社区/有署名的专家
- [ ] **发布日期**: 在 2 年内（经典基础理论除外，但需标注版本兼容性）
- [ ] **内容完整度**: 不是片段、摘要或广告
- [ ] **是否有过时信息**: 检查版本号、API 是否已废弃
- [ ] **格式合适**: 文章/视频/文档/播客，匹配用户需求

### 验证方法
1. 用 `web_fetch` 抓取页面，检查头部信息（作者、日期、版本）
2. 官方文档检查 URL 域名（如 `react.dev` 而非某个 blog 域名）
3. GitHub 仓库检查 last commit 时间
4. 技术博客检查评论区是否有「这篇内容已过时」的提示

## 拒绝来源（黑名单）

- ❌ 无署名的搬运号/聚合号
- ❌ 超过 3 年未更新且涉及快速迭代技术的内容
- ❌ 明显的营销软文（通篇推销产品/课程）
- ❌ 翻译质量差的机翻文章
- ❌ 论坛里零回复、零点赞的帖子

## 搜索失败回退

如果某个维度搜索结果不足：
1. 扩大搜索词范围（去掉版本年份限制）
2. 尝试同义词/别名搜索
3. 用英文搜索词搜索英文权威内容
4. 最多降级一次质量阈值，不要推荐明显过时内容
