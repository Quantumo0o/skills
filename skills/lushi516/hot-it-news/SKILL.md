# hot-it-news

## 何时使用

当用户提出类似需求时使用：

- 获取今天热门 IT 资讯
- 获取量子位、人人都是产品经理、虎嗅热门文章
- 汇总 AI / 产品 / 商业科技热点
- 输出一份业内热文清单或晨报

## 核心原则

这是一个**以站点首页明确热门区块为准**的资讯技能，不是搜索引擎拼凑技能。

必须遵守：

- **优先直接打开站点首页**，读取其明确的热门/推荐区块
- **默认输出的每条资讯必须包含真实文章链接**
- **不编造链接，不冒充链接，不用首页/栏目页冒充文章页**
- **不能把搜索结果、最近更新、普通列表页混充为站内热门榜单**
- **如果未能确认指定区块，就必须明确说明，不得伪装成已严格命中目标区块**

## 目标站点与正确获取方式

不要优先依赖搜索引擎猜测热门内容。
优先直接打开站点首页，读取其明确的热门区域。

### 1) 量子位
- 网址：`https://www.qbitai.com/`
- 获取方式：打开首页，读取**右侧「热门文章」**区块
- 说明：这是站内明确展示的热门文章入口，优先级高于搜索结果

### 2) 人人都是产品经理
- 网址：`https://www.woshipm.com/`
- 获取方式：打开首页，读取**右侧「推荐文章」**区块
- 说明：不要先靠搜索猜热文，优先取首页右侧推荐内容

### 3) 虎嗅
- 网址：`https://www.huxiu.com/`
- 获取方式：打开首页，读取**「48小时热文」**区块
- 说明：该区块是虎嗅首页可直接确认的热文来源

## 推荐工作流

### Step 1：打开首页
优先使用浏览器工具分别打开：
- `https://www.qbitai.com/`
- `https://www.woshipm.com/`
- `https://www.huxiu.com/`

### Step 2：定位目标区块
必须优先确认 skill 指定区块是否真的被读取到：
- 量子位：首页右侧「热门文章」
- 人人都是产品经理：首页右侧「推荐文章」
- 虎嗅：首页「48小时热文」

如果页面抓取不稳定：
- 先尝试更可靠的浏览器读取方式
- 再尝试补抓链接
- 不要因为抓取困难就直接退化成“首页可见内容随便摘几条”而不说明

### Step 3：提取字段
从目标区块中提取：
- 标题
- **文章真实链接**
- 如页面中可见，顺带提取时间或简单标签

### Step 4：结果校验
输出前必须逐条自检：
- 是否有标题
- 是否有**真实文章链接**
- 链接是否指向具体文章页，而不是首页/栏目页/标签页
- 内容是否来自指定热门区块，而不是普通信息流

## 链接规则（强制）

### 默认要求
**默认返回的每条资讯必须包含真实文章链接。**

也就是说：
- 允许输出：`标题 + 真实文章链接`
- 不允许把“只有标题”当作正式完成结果

### 拿不到链接时怎么办
如果标题确认了，但真实文章链接暂时拿不到：

1. **先继续补抓链接**，不要立刻输出残缺结果
2. 如果仍无法确认，必须明确标注：
   - `标题已确认，链接待补全`
3. 这种情况应被视为**异常降级**，不是默认输出形态
4. 除非用户明确接受“仅标题版”或“先看速览版”，否则**不应把无链接结果当正式结果提交**

### 低质量结果处理
如果某个站拿到的有效条目（标题 + 真实链接）少于 3 条：
- 必须明确说明该站抓取不完整
- 不要伪装成完整榜单
- 宁可少报，也不要凑数

## 输出格式

默认输出为你指定的固定格式：

```markdown
以下是今天业内热门文章

# 量子位 - 热门文章

## 1. [标题](link)
## 2. [标题](link)
## 3. [标题](link)

# 人人都是产品 - 推荐文章

## 1. [标题](link)
## 2. [标题](link)
## 3. [标题](link)

# 虎嗅 - 48 小时热文

## 1. [标题](link)
## 2. [标题](link)
## 3. [标题](link)
```

要求：
- 顶部固定写：`以下是今天业内热门文章`
- 一级标题固定为：
  - `# 量子位 - 热门文章`
  - `# 人人都是产品 - 推荐文章`
  - `# 虎嗅 - 48 小时热文`
- 每篇文章必须使用二级标题格式：
  - `## 1. [标题](link)`
- **正式结果中，不允许只输出标题不输出链接**
- 不额外添加摘要，除非用户明确要求
- 不要输出多余说明文字，除非用户追问来源、准确性，或某站抓取不完整

## 降级输出格式（仅异常时使用）

只有在以下情况才允许降级：
- 页面结构异常
- 指定区块无法稳定读取
- 标题可确认但真实文章链接无法补全
- 用户明确同意先看速览版

降级时必须显式说明，不得伪装成正式版。例如：

```markdown
以下是今天业内热门文章（部分链接待补全）

# 量子位 - 热门文章

## 1. 标题已确认，链接待补全：xxx
## 2. [标题](link)
## 3. [标题](link)
```

## 扩展格式（仅在用户明确要求时使用）

```markdown
今日 IT / AI / 产品 热门文章

一、量子位
1. 标题
   链接：...
   一句话：...

二、人人都是产品经理
1. 标题
   链接：...
   一句话：...

三、虎嗅
1. 标题
   链接：...
   一句话：...
```

扩展格式也必须遵守：
- 默认每条都有链接
- 摘要是一句话补充，不可替代链接

## 风格要求

- 输出简洁、好读、可直接转发
- 不要写花哨废话
- 如果用户没要求摘要，就先给标题+链接
- 如果内容很多，优先每站 5~8 篇
- 如用户强调“今天”，要明确说明：
  - 量子位：基于首页右侧热门文章
  - 人人都是产品经理：基于首页右侧推荐文章
  - 虎嗅：基于首页48小时热文

## 严谨性要求

如果用户追问“你怎么获取的”“是不是今天热门”：
要明确回答：
- 量子位来源于首页右侧热门文章
- 人人都是产品经理来源于首页右侧推荐文章
- 虎嗅来源于首页48小时热文

不要把这三者混称为完全同一种榜单：
- 量子位：更接近站内热门文章
- 人人都是产品经理：更接近首页推荐文章
- 虎嗅：明确是48小时热文

如果本次抓取**没有严格命中指定区块**，必须明确说：
- 当前结果是首页热点整理 / 近似提取
- 不是严格意义上的指定榜单复刻

## 禁忌

不要：
- 先用搜索结果拼凑“热门榜”
- 用栏目页替代文章页
- 在链接不确定时假装确定
- 把“最近更新”说成“今日热榜”
- 把“标题列表”伪装成正式完成结果
- 在未确认指定区块时，谎称已按 skill 严格抓取完成

## 可选增强

如果用户要更完整内容，可进一步：
- 给每篇补一句摘要
- 合并为“今日晨报”
- 按 AI / 产品 / 商业科技 三类重新归类
- 在结尾补充“抓取说明/来源说明”以方便转发时自证来源
