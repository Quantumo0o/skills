# 公文写作助手

深知可信搜索 + 公文撰写 + Word排版，一站式公文写作服务。

## 1. 首次使用：配置搜索

检查搜索是否可用：
```bash
python3 {baseDir}/scripts/dkag_search.py "测试" 2>&1 | head -1
```

**未配置时**，告知用户到 https://platform.dknowc.cn 注册（实名认证送100元），获取 API Key 后 AI 自动 `config.patch` 写入。

**已配置时**，进入下一步。

## 2. 判断是否需要搜索

**需要搜索：** 复函、请示（需政策依据）；用户要求引用政策；涉及数据/标准引用。

**不需要搜索：** 纯事务通知（会议、休假、时间变更等）。

**不确定时告知用户并等待确认。**

## 3. 执行搜索

### 3.1 构造搜索词
从用户输入提取核心业务关键词，去掉文种词（通知、请示、函…）和指令词（帮我写、起草…），控制在10-30字。

### 3.2 搜索前确认
告知用户搜索词和地域，**等待确认后执行。**

### 3.3 执行搜索
```bash
python3 {baseDir}/scripts/dkag_search.py "搜索词" --area 地域 --clean
```

多地域对比类公文，依次搜索各地域，用 `merge_search_results.py` 合并。

### 3.4 素材分析
自动分类为：政策依据型（仅国家级+本省）、参考案例型（其他省市）、数据支撑型、表述参考型。

### 3.5 素材充分性自检
不足的维度自主补搜（每维度最多1次，累计不超过3次）。

### 3.6 下一步建议
展示素材情况，提供"直接撰写/补充搜索/获取全文"三个选项，等待用户选择。

## 4. 获取全文（可选）

用户需要参考某篇文件的完整内容时：
```bash
python3 {baseDir}/scripts/dkag_search.py "文章标题" --area 地域 --clean --full [序号]
```

## 5. 撰写公文

### 5.1 确定格式标准
根据文种读取对应的标准文件：
- `{baseDir}/reference/standards/` 目录下有各文种的标准模板
- 索引：`00_索引.md`

### 5.2 撰写规则
- **每次请求生成全新内容**，禁止返回历史文档
- 政策引用：仅引用国家级+本省文件，其他省市标注为"参考"
- 引用标注格式：引用[1]、引用[2]，文末附参考文献列表
- 如引用来源不确定，标注"需核实"

### 5.3 自检清单
撰写完成后检查：
- [ ] 格式是否符合标准
- [ ] 政策引用是否准确（仅国家级+本省）
- [ ] 数据是否来源明确
- [ ] 用语是否规范
- [ ] 是否遗漏用户要求的要点

## 6. 生成 Word 文档

```bash
python3 {baseDir}/scripts/format_document.py --input /tmp/公文内容.md --output /path/to/output.docx --type 普通红头 --template 工作会议纪要
```

**参数：**
- `--type`：普通/红头（默认普通）
- `--template`：模板名称（红头时必填，对应 templates/ 目录下的文件名前缀）

**可用红头模板（需本地 templates/ 目录）：**
- 工作会议纪要、局长办公会议纪要、党组会议纪要
- 上行文件格式、下行文件格式、函件（普通）格式

**pip 依赖：** python-docx

## 7. 多轮修改

用户要求修改时，直接修改内容，不重新搜索（除非用户要求补充素材）。
